SME Ops System — 薪資系統完整規格

v3（補強完成版）　·　產出 2026-05-05　·　核心原則：所有計算邏輯在 DB function generate_payroll，前端只觸發/顯示

A. 系統定位

SME Ops System 薪資模組以 PostgreSQL function 為計算核心，遵循三條設計原則：

  1. 單一事實來源（SSoT）：薪資結構、級距、法令率全部存在 DB master tables，前端不複寫邏輯
  2. 計算集中：每月一個 function 呼叫做完所有計算 → 寫入 payroll_records → 前端讀
  3. 稽核可追溯：每筆計算的 breakdown（自訂津貼、法扣、二代健保） 都用 JSONB 保留明細

對外連動：勞保局（提繳清冊）、健保署（補充保費申報）、國稅局（扣繳憑單）。

C. GROSS 計算（收入項）

項目計算方式
底薪 base_salary月薪：salary_structures.base_salary 直接帶入
時薪：hourly_rate × attendance_records.total_hours 累加
固定加給role_allowance + meal_allowance + transport_allowance（直接帶入）
自訂津貼 custom_allowancesSUM(JSONB 陣列中所有 amount)　例：[{"name":"夜班","amount":3000}, {"name":"證照","amount":500}] → 3500
全勤獎金 attendance_bonussalary_structures.attendance_bonus，但該月遲到分鐘 > 0 OR 請假天數 > 0 → 直接清零
加班費 overtime_pay見 D 區（勞基法分級）
未休特休折現 unused_leave_payout僅離職員工：抓 leave_balances 未休特休天數 × daily_rate（base_salary / 工作天數）
年終獎金 year_end_bonus獨立 function generate_year_end_bonus 觸發，pay_period='YYYY-13'
GROSS= 底薪 + 加給 + 自訂津貼 + 全勤 + 加班費 + 未休假折現

D. 加班費分級（勞基法 §24/§32 完整級距）

類型時段倍率（時薪 = 底薪 ÷ 工作天 ÷ 8）
平日加班 §24-1前 2 小時時薪 × 1.34（4/3 加給）
第 3 小時起時薪 × 1.67（5/3 加給）
休息日加班 §24-20–2 小時時薪 × 1.34
第 3–8 小時時薪 × 1.67
第 9 小時起時薪 × 2.67（8/3 加給）

每月加班上限 §32：一般 46 小時、有變形協議者 54 小時、3 個月不超過 138 小時。本系統 MONTHLY_OVERTIME_CAP = 46 hours。

F. 扣項計算

項目計算方式
請假扣薪無薪假 = 天數 × daily_rate；半薪假 = 天數 × daily_rate × 0.5；全薪不扣
遲到扣薪FLOOR(累計遲到分鐘 / 30) × (hourly_rate × 0.5)
勞保員工自付從 labor_ins_brackets 抓 employee_premium（按 employees.labor_ins_grade 與年度）
健保員工自付從 health_ins_brackets 抓 employee_premium × (1 + 眷屬數)
勞退員工自願自提LEAST(base, 150000) × (employees.labor_pension_self_rate / 100)
所得稅扣繳呼叫 _calc_monthly_withholding(gross)：5 級累進（見 G）
二代健保補充保費2 種觸發：① 加班費 > 月投保金額 → 超出部分 × 2.11%；② 年累計獎金 > 投保 × 4 → 超出部分 × 2.11%
法扣 fixed 模式每月固定 monthly_amount
法扣 percent 模式每月 GROSS × monthly_percent / 100
NET 實領= GROSS − 所有扣項　如扣前餘額 < 法扣應扣，部分扣 + 自動延後到下月

I. 法扣兩種模式（v2 新）

模式欄位說明 / 適用場景
fixed 固定金額monthly_amount NUMERIC每月固定扣 X 元，扣到 paid_amount=total_amount 為止。
適用：信用卡欠款、養育費固定金額。
percent 月薪百分比monthly_percent NUMERIC(5,2)每月扣 GROSS × X%，金額隨月薪浮動。
適用：法院強制扣薪「薪資 1/3」(33.33%)、25% 之類。

DB 用 CHECK constraint 強制兩種模式互斥（fixed 必填 monthly_amount、percent 必填 monthly_percent）。

J. 完整計算流程圖

每月 admin 進 [HR → 薪資發放] 點「新增薪資作業」（YYYY-MM） ↓ DB 端 generate_payroll('2026-05', created_by) 觸發 ↓ INSERT payroll_runs (status='draft') ↓ FOR 每位 (status='在職' OR (status='離職' AND resign_date 在當月)) DO ↓ 讀 master：salary_structures, employees, labor_ins_brackets, health_ins_brackets 讀 transactional：attendance_records, leave_requests, overtime_requests, leave_balances ↓ 【收入項】 base_salary（月/時薪）+ allowances + custom_allowances + attendance_bonus（遲到/請假則 0） + overtime_pay（平日 1.34/1.67、例假 1.34/1.67/2.67） + unused_leave_payout（僅離職員工） → GROSS ↓ 【二代健保補充保費】 IF 加班費 > 月投保金額 → 超額 × 2.11% IF 年累計獎金 > 投保 × 4 → 超額 × 2.11% → 寫 nhi_supplementary_records 明細表 ↓ 【扣項】 - 請假扣（payRate 0/0.5） - 遲到扣（30 分鐘 × 半小時時薪） - 勞保自付 / 健保自付 (× 眷屬乘數) - 勞退員工自願自提（base × self_rate / 100） - 所得稅 5 級累進 - 二代健保補充保費 - 法扣（fixed 或 percent，按進行中順序扣到餘額用完） ↓ INSERT payroll_records（含全部 breakdown JSONB） ↓ END LOOP ↓ admin 審閱 → 「定案」按鈕 → status='finalized' ↓ 觸發 hr.salary.calculated 事件 → 推 LINE 通知 → 員工查薪資單

K. 計算範例（5 個典型情境）

K1. 一般月薪員工

員工 A：base 30,000、role 2,000、meal 500、夜班自訂 3,000、全勤 1,000 本月狀況：請 2 天事假、遲到 60 分鐘、平日加班 4h、例假加班 6h hourly_rate = 30,000 / 30 / 8 = 125 【GROSS】 base + 加給 + 自訂 = 35,500 全勤 = 0 (有遲到/請假清零) 平日 OT 2×1.34×125 + 2×1.67×125 = 752.5 例假 OT 2×1.34×125 + 4×1.67×125 = 1,170 GROSS = 37,422 【扣項】 事假扣 2 × 1,000 = 2,000 遲到扣 FLOOR(60/30)×62.5 = 125 勞保 grade 14 = 714 健保 452 × (1 + 0) = 452 勞退自提 0% = 0 所得稅 37,422 ≤ 40,020 = 0 二代健保 OT 1,170 ≤ 投保金額 = 0 法扣 = 0 扣項合計 = 3,291 【NET】= 37,422 − 3,291 = 34,131

K2. 高所得 + 觸發二代健保（加班費）

員工 B：base 50,000、grade 21（投保 50,600） 本月平日加班 60h（極端範例） hourly_rate = 50,000 / 30 / 8 ≈ 208 平日 OT 2×1.34×208 + 58×1.67×208 = 557.4 + 20,142.9 ≈ 20,700 GROSS = 50,000 + 加給 + 20,700 ≈ 72,000 【二代健保補充保費】 OT 20,700 < 50,600 → 0 （未觸發，因加班費未超月投保） 但若 OT = 60,000： 超額 60,000 − 50,600 = 9,400 扣繳 9,400 × 2.11% = 198 元 → 寫 nhi_supplementary_records (category='加班費超額')

K3. 離職結算

員工 C：5 月底離職，當月特休餘額 8 天未休 base 36,000，工作天 22 天 → daily_rate = 1,636 generate_payroll('2026-05') 對象擴大： status='離職' AND resign_date IN [05-01, 05-31] 【加項】 unused_leave_payout = 8 × 1,636 ≈ 13,088 → 加進 GROSS DB trigger trg_resign_cascade 不觸發（已是離職狀態） 但 payroll_records.is_final_settlement = true 標記

K4. 年終獎金 + 觸發二代健保（4 倍累計）

員工 D：base 60,000、grade 23（投保 60,800） 年度累計獎金已達 200,000（前 11 個月 attendance_bonus 累計） generate_year_end_bonus(2026, NULL) 跑： 年終 = 60,000 × 2 個月 = 120,000 【4 倍門檻】= 60,800 × 4 = 243,200 新累計 = 200,000 + 120,000 = 320,000 超出 = 320,000 − 243,200 = 76,800 補充保費 = 76,800 × 2.11% = 1,621 【NET 年終】= 120,000 − 所得稅 − 1,621 = ? 所得稅 = 11,399 + (120,000 − 120,000) × 30% = 11,399 NET = 120,000 − 11,399 − 1,621 = 106,980

K5. 法扣 percent 模式

員工 E：法院強制扣薪「薪資 1/3」上限 deduction_type='percent', monthly_percent=33.33 5 月 GROSS = 50,000 → 應扣 50,000 × 33.33% = 16,665 6 月 GROSS = 60,000 → 應扣 60,000 × 33.33% = 19,998 扣到 paid_amount = total_amount 才停止 （用月薪百分比讓扣款隨薪資波動，員工生活仍有保障）

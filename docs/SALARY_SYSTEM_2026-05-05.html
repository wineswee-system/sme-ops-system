A. 系統架構總覽

核心設計原則：所有薪資計算邏輯都在 DB 端（PostgreSQL function），前端只負責「觸發 + 顯示」。每月在 [HR → 薪資發放] 頁面點「結算」 → 呼叫 generate_payroll('YYYY-MM') → DB 對全體員工跑一次計算 → 寫入 payroll_records 表。

月初 admin 觸發 ↓ generate_payroll('2026-05') ← SQL function（Postgres） ↓ FOR 每位 (status='在職' OR 當月離職) 員工 LOOP ← v2 新：含當月離職 讀 salary_structures（薪資組成） 讀 attendance_records（打卡 / 遲到分鐘） 讀 leave_requests（假別 + 天數） 讀 overtime_requests（加班時數 + 平日/例假） 讀 labor_ins_brackets / health_ins_brackets（勞健保級距） 讀 legal_deductions（法扣進度） 讀 leave_balances（特休餘額 → v2 新：離職員工折現） 讀 annual_bonus_tracker（年度獎金累計 → v2 新：二代健保 4 倍門檻） ↓ GROSS（總薪）= 底薪 + 各加給 + 加班費 + 全勤獎金 + 未休假折現(離職) 扣項 = 請假扣 + 遲到扣 + 勞保 + 健保 + 所得稅 + 法扣 + 二代健保補充保費(v2新) + 勞退員工自提(v2新) NET（實領）= GROSS − 扣項 ↓ INSERT payroll_records END LOOP ↓ 產出 payroll_runs（草稿） → admin 簽核 → 已定案 → 匯出銀行清冊

C. 計算流程（generate_payroll 主函式）

C1. 收入項（GROSS）

項目計算方式
底薪salary_structures.base_salary（月薪）／hourly_rate × 時數（時薪制）
固定加給role_allowance（職級）+ meal_allowance（膳食）+ transport_allowance（交通）
自訂津貼SUM(custom_allowances JSONB 陣列中所有 amount)　例：夜班 3000 + 證照 500 + 主管加給 5000
全勤獎金attendance_bonus　但該月若有任一筆遲到 OR 任一天請假 → 直接清零
加班費見 D 區（勞基法分級）
GROSS（總薪）= 底薪 + 固定加給 + 自訂津貼 + 全勤 + 加班費

C2. 扣項（DEDUCTIONS）

項目計算方式
請假扣薪FOR 每筆請假 → 天數 × daily_rate × (1 − payRate)
例：請 2 天事假（payRate=0）→ 扣 2 × daily_rate × 1.0 = 2 天滿薪；請 2 天病假（payRate=0.5）→ 扣 2 × daily_rate × 0.5 = 1 天薪
遲到扣薪FLOOR(late_minutes / 30) × (hourly_rate × 0.5)
遲到每滿 30 分鐘扣半小時薪
勞保自付從 labor_ins_brackets 抓 employee_premium（按員工 labor_ins_grade）
2026 年費率 13%（普通 12% + 就業險 1%），員工分擔 20%（= 2.6%）
健保自付從 health_ins_brackets 抓 employee_premium × (1 + 眷屬數)
2026 年費率 5.17%，員工分擔 30%。眷屬乘數：員工自己 1 份 + 每名眷屬 1 份
勞退（員工自提）目前 = 0（欄位有預留，邏輯尚未實作）
註：雇主強制提繳 6% 由公司付，不在這裡扣
所得稅扣繳呼叫 _calc_monthly_withholding(gross) 函式（見 E 區）
法扣FOR 每筆 legal_deductions WHERE status='進行中'
應扣 = MIN(monthly_amount, 剩餘總額, 扣前餘額)
扣到 paid_amount = total_amount → status='已完成'
未扣完自動延後下個月
NET（實領）= GROSS − (請假扣 + 遲到扣 + 勞保 + 健保 + 勞退 + 稅 + 法扣)

I. 計算範例（以一名員工試算 2026-04 月薪）

員工 A：月薪制，base 30,000、role 2,000、meal 500、自訂夜班津貼 3,000、全勤 1,000 本月狀況：請 2 天事假、遲到 60 分鐘、平日加班 4 小時 + 例假加班 6 小時 hourly_rate = 30,000 / 30 / 8 = 125（簡化） 【GROSS】 base = 30,000 + role + meal = 2,500 + 夜班自訂 = 3,000 + 全勤 = 0 （有遲到/請假 → 清零） + 平日 OT 2h×1.34×125 + 2h×1.67×125 = 335 + 417.5 = 752.5 + 例假 OT 2h×1.34×125 + 4h×1.67×125 = 335 + 835 = 1,170 ───────────────────── GROSS = 37,422.5 【扣項】 − 事假扣 2 × (30,000/30) = 2,000 − 遲到扣 FLOOR(60/30) × (125×0.5) = 2 × 62.5 = 125 − 勞保 從 grade 14 表抓 = 714 − 健保 452 × (1 + 0 眷屬) = 452 − 勞退自提 0 （邏輯未實作） − 所得稅 37,422 ≤ 40,020 = 0 − 法扣 無進行中紀錄 = 0 ───────────────────── 扣項合計 = 3,291 【NET 實領】= 37,422.5 − 3,291 ≈ 34,132

K. v2 補強內容（2026-05-05 完成）

1. 二代健保補充保費 ✅已加入 generate_payroll：(a) 加班費超月投保金額部分扣 2.11%；(b) 獎金累計超 4 倍級距扣 2.11%。明細記錄於 nhi_supplementary_records，可從 v_nhi_supplementary_filing 匯出申報 CSV。
2. 離職結算 ✅generate_payroll 對象擴大為「在職 OR 當月離職」。離職員工最後當月自動結算未休完特休折現（從 leave_balances 抓）→ 寫入 payroll_records.unused_leave_payout，並標記 is_final_settlement=true。
3. 勞退員工自願自提 ✅employees 加 labor_pension_self_rate (0~6%)。generate_payroll 自動扣繳：員工自提 = base × rate / 100，封頂 150,000。
4. 年終獎金結算 ✅新 function generate_year_end_bonus(year, months_override)。pay_period='YYYY-13' 表示年終批次。Payroll.jsx 頁首加按鈕觸發。
5. 申報視圖 ×3 ✅v_labor_pension_filing_monthly（勞退提繳清冊）、v_nhi_supplementary_filing（補充保費明細）、v_payroll_summary_monthly（月度總表）。Payroll.jsx 加 CSV 匯出按鈕。

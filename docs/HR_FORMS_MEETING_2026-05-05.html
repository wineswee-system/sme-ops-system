HR 表單系統說明 SME Ops System · 開會用簡報　·　2026-05-05

📌 一句話總結

員工進「HR 表單中心」一站式申請所有 HR 流程，分 4 大類共 18 種表單；admin 可在「表單建立器」拉欄位自建新表單。送單後依設定走簽核鏈或自動推給直屬主管，全程通知中心 + LINE 推播。

🔀 兩種核准方式

自動解析 適合日常 HR簽核鏈 適合重大決策
系統自動找：店長 → 直屬主管 → 部門主管 → 部門樹往上預先設定 N 關步驟，每關指定簽核人
申請人是店長本人 → 走 HR 權限者跑完所有關才算通過
用於：請假、加班、補登、出差、報帳等用於：離職、留停、異動、費用申請等
不用 admin 預先設定需 admin 進「簽核鏈設定」建立
📅 第一類：假勤申請（5 種）

📝請假申請　自動解析

填什麼假別（13 種）、起訖日、原因、附件（診斷書等）
核准直屬主管
影響批准後 → 排班跳過該日 → 薪資依假別自動扣（全薪/半薪/無薪）→ 扣特休餘額
特色員工餘額不足前端會擋下；事假無薪、病假前 30 天半薪、特休全薪

⏰加班申請（事後補登）　自動解析

填什麼加班日、起訖時間、加班時數、原因、補休 or 加班費
核准直屬主管
影響批准後 → 加班費自動進入薪資計算 → 觸發二代健保補充保費判定

📋加班申請（事前預先）　自動解析 表單建立器自訂

填什麼同事後加班，但需「預先」申請
為什麼勞基法 §32 要求加班需事先取得員工同意，事前申請是合規證明
勞檢勞工局稽查必查項目，沒做會被開單

🕐忘刷補登　自動解析

填什麼員工、日期、補登類型（上班/下班）、實際打卡時間、原因
核准直屬主管
影響批准後 → 自動補進打卡紀錄 → 影響遲到計算 / 工時 / 薪資

↩️銷假申請　自動解析 表單建立器自訂

填什麼原假單、銷假原因
核准直屬主管
影響批准後 → 還原該員工特休餘額、薪資扣薪自動調整
🏃 第二類：人事異動（5 種）

🚪離職申請　簽核鏈

填什麼員工、預計離職日、原因（個人/家庭/健康/另謀高就/退休）、原因說明、交接事項
核准直屬主管 → HR 確認 → 執行長核准（3 關）
影響最後一關核准 → 員工狀態自動改「離職」+ 寫離職日 → 下個月薪資自動進入「離職結算」（含未休特休折現）
隱私非 admin 只能看自己的申請

⏸️留職停薪　簽核鏈 表單建立器自訂

填什麼員工、起迄日、預計回任日、原因類型（產假/育嬰/兵役/進修/家庭）、附件
核准直屬主管 → HR 確認 → 執行長核准（3 關）
影響核准 → 員工狀態自動改「留停」；填回任日後 → 自動改回「在職」
備註育嬰留停依性平法為法定權利，HR 不可拒

🔄人事異動申請　簽核鏈

填什麼員工、異動類型（調職/升遷/降調/部門調動/跨店調動/調薪）、生效日、新部門、新門市、新職位、新薪資、原因
核准直屬主管 → HR 確認 → 執行長核准（3 關）
影響最後一關核准 → 自動寫異動軌跡 + 更新員工主檔（含底薪）→ 下月薪資套新數字
UI 特色顯示異動前 / 後對比（劃掉舊值、紅字顯示新值）

⏳試用期評核　自動解析

填什麼員工、指導人、起訖日、評分項目、結果（試用中/已通過/未通過/延長）、評語
影響未通過 → 啟動離職流程；已通過 → 員工正式入職、加入正常排班輪
特色新員工到職後 90 天系統自動建立試用記錄

⭐績效考核　簽核鏈

填什麼員工、評核期間、自評、主管評、整體分數、KPI 項目、改進方向、下期目標
核准主管 → HR 雙簽
影響結果可帶入績效獎金計算；連續 2 期低分 → AI 離職預測加權重
💰 第三類：費用 / 出差（3 種）

✈️出差申請　自動解析

填什麼出差地點、起訖日、目的、預估費用、交通方式
核准直屬主管
影響批准後 → 排班跳過該日（不被排店內班） → 可申請費用報銷

🧾費用報銷　自動解析

填什麼類別（交通/住宿/餐費/雜支）、金額、收據（上傳）、關聯出差單、說明
核准直屬主管 → 財務
影響批准後 status='已核銷' → 進入會計流程

📑費用申請（採購類）　簽核鏈（金額分流）

填什麼標題、說明、預估金額、會計科目、門市、附件
金額分流 ≤ 3,000：直屬主管 1 關
3,001–10,000：主管 + 部門主管 2 關
≥ 10,001：主管 + 部門主管 + 財務 3 關
影響核准後 → 進入採購流程；可實際撥款後核銷
📋 第四類：其他（5 種）

👥招募職缺申請　簽核鏈

填什麼部門、職稱、人數、薪資範圍、急迫性、原因、應徵資格
核准部門主管 → HR → 執行長
影響核准後 → 可外發 104 / 開始接收履歷

🎓教育訓練　自動解析

填什麼課程名稱、單位（內訓/外訓）、起訖日、費用、補助比例、結業狀態
核准主管核准 + HR 報名
影響結業後 → 寫入員工證照清單；費用核銷走費用報銷流程

📄文件簽核　簽核鏈（依文件類型）

填什麼標題、文件類型（合約/政策/協議書/通知）、檔案、相關員工、生效日、到期日
核准合約走法務、政策走 HR、其他依設定
影響員工簽核後 → 寫個人簽收記錄

🚑工傷通報　自動解析

填什麼事故日、地點、類型（職災/通勤/其他）、傷勢、就醫情況、是否已通報勞工局
核准主管 → HR 立即處理（24h 內）
影響嚴重案件 → 自動產勞工局通報書 PDF；薪資算「公傷病假」全薪
法令勞基法 §59：職災原領工資補償

💬員工意見反映 / 滿意度調查　無核准

填什麼調查題目（題型多樣）、回應內容、整體分數、評論
特色匿名收集（無需核准）
影響低分 → 推 HR；連續低分員工 → AI 離職預測加權重

🛠️ 表單建立器（Form Builder）

admin 不用寫程式就能新增表單。適合輕量級需求（如：員工健康申報、設備領用、活動報名）。

7 種欄位類型

類型表現方式適用
單行文字單行輸入框姓名、項目名稱、簡短描述
多行文字多行輸入框說明、原因、備註
數字數字輸入金額、數量、天數
日期日期選擇器申請日、生效日、截止日
下拉選擇單選下拉類別、狀態、優先級
多選勾選多選 checkbox多項目選擇
檔案上傳檔案附件診斷書、合約、收據

建表單時可以設

  • 表單名稱　顯示在 HR 表單中心的卡片
  • 分類　歸入哪一大類（假勤/人事異動/費用/其他）
  • 顏色　卡片圖示顏色（7 色可選）
  • 欄位　拉幾個欄位都行（每個可設必填、預設值、選項）
  • 簽核鏈　指定走哪條簽核鏈（可空 = 不簽核直接通過）
  • 啟用 / 停用　不啟用就不顯示在表單中心
  • 排序　在類別內的顯示順序

🔗 簽核鏈範本（4 條 HR 標準鏈）

進「簽核鏈設定」點「✨ HR 範本」一鍵建立：

鏈名類別步驟
離職申請離職申請人 → 直屬主管 → HR 確認 → 執行長核准 → ✓
留停申請留停申請人 → 直屬主管 → HR 確認 → 執行長核准 → ✓
人事異動異動申請人 → 直屬主管 → HR 確認 → 執行長核准 → ✓
預先加班預先加班申請人 → 直屬主管 → 部門主管 → ✓
建立後請逐條編輯，把每個 step 的「簽核人」用搜尋下拉指派具體員工。否則送單會卡在「找不到第一關簽核人」。

🔄 簽核流程（通用機制）

1員工送單　→ 系統自動找第一關簽核人
▼
2通知簽核人　→ 寫進通知中心 + LINE 推播
▼
3簽核人決定　→ 核准 / 駁回（需填原因）
▼
4核准 → 推下一關，駁回 → 退回申請人改後重送
▼
5最後一關通過　→ 觸發後續動作（員工狀態改、薪資調整、寫軌跡）

UI 顯示

  • 列表頁顯示「簽核進度」：綠 = 已過 橘 = 當前 灰 = 等待
  • 表單上方顯示「· 簽核鏈：xxx（N 關）」或「⚠ 無簽核鏈」警示
  • 只有當前 step 的合法簽核人才看得到「核准 / 駁回」按鈕

💡 開會討論重點

  1. 簽核鏈每關的具體簽核人　離職/留停/異動的 3 關要選誰（直屬主管 / HR / 執行長）
  2. 銷假 / 留停 / 預先加班的欄位　決定要設哪些欄位再用建立器拉
  3. 費用申請的金額分級　3,000 / 10,000 / 10,000 以上的門檻是否合理？
  4. 看其他人提交的權限　主管能否看部門內所有人的提交？
  5. LINE 通知是否啟用　每張表單都要 LINE 推播還是只有重要的？
  6. 表單建立器自訂欄位的命名規範　建議統一格式（如：「日期類欄位用 YYYY-MM-DD」）

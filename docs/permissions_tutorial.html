1. 權限制度速覽

本系統採用 5 角色 + 個別覆寫 制度。每位員工歸屬一個角色，角色帶有預設權限；如果某個員工需要例外，再「個別調整」覆寫掉角色預設。

5 個角色

角色定位典型權限
super_admin最高層所有功能
admin系統管理員HR / 經費 / 流程 全部能改
manager主管 / 店長HR 簽核 / 排班核准 / 看自己單位資料
office_staff辦公室員工看大部分資訊、改少數（如自己的請假申請）
store_staff門市員工進員工 Portal、看薪資 / 班表、提申請

權限怎麼算？

  1. 系統先看員工的「角色預設」 → 拿到一份預設權限清單
  2. 再看員工的「個別調整」（如果有的話）→ 覆蓋掉預設
  3. 結果就是該員工真正擁有的權限
舉例： 小華是 store_staff，角色預設沒有「員工資料修改」。但 admin 給他「個別加給」這個權限，他就能改員工資料。如果 admin 把他從 office_staff 改回 store_staff，這個個別調整還在，他仍保有這個權限（直到 admin 把它移除）。

2. 怎麼進到權限設定頁

1用 admin 或 super_admin 帳號登入
2左側 sidebar → 系統設定 → 權限設定（或直接打開 /system/employee-permissions）
3左側列出所有員工，點要設定的員工 → 右側顯示該員工所有 module 的權限明細

3. 畫面解讀：「查詢」「修改」兩顆按鈕

每個功能（feature）右側都有兩顆按鈕：

按鈕意思關掉時
✓ 查詢 看得到資料但不能改 連功能都看不到
✓ 修改 看得到 + 可以改 只能查詢、不能改動
小提醒： 按鈕有連動邏輯：
  • 點「修改」開 → 自動把「查詢」也開（不可能能改但看不到）
  • 點「查詢」關 → 自動把「修改」也關（看不到怎麼改）

4. 狀態色碼說明

每個功能旁邊會有個小標籤，告訴你這個權限是「哪裡來的」：

標籤含義什麼時候會出現
角色預設 沒做個別調整，照角色預設來 絕大多數情況
手動調整（綠色） 角色預設沒給，但個別加給了 例：給某個門市員工特權看薪資
手動調整（紅色） 角色預設有，但個別禁用了 例：某個主管暫時不能改員工資料

有手動調整的 row 會用對應顏色框起來，旁邊還有 ↻ 圖示 — 點它就會把調整移除、回到角色預設。

5. 常見操作場景

情境 A：給某員工加額外權限

1左側找到該員工，點選
2找到對應的功能 row（例：HR / 員工管理 / 修改）
3點「修改」按鈕 → 變青色（按鈕亮起 + ✓）
4row 旁邊會出現 手動調整 綠色標籤，整 row 加綠框

情境 B：收回某員工的個別權限

1找到該員工 → 找到被「個別加給」的功能（綠標籤）
2有兩種方式：
  • 方式 A： 點該功能的「修改」按鈕關掉 → 變紅色標籤 手動調整（角色預設有但被禁用，僅限角色本來就有的情況）
  • 方式 B： 點 row 最右側的 ↻ 圖示 → 整個移除個別調整、回到角色預設

情境 C：全部恢復角色預設（清零）

1找到該員工
2右上角有一顆「↻ 全部恢復角色預設」按鈕
3確認後會清除這個員工所有個別權限調整，所有 row 回到 角色預設 狀態
注意： 此操作不可逆。建議先記錄該員工目前的個別調整有哪些（看畫面上的綠 / 紅 row），再操作。

情境 D：直接改員工角色（更大刀的方式）

有時候員工換職務，整個角色換掉比較快。例：office_staff 升 manager。

1到 系統設定 → 員工管理（或直接編輯員工資料）
2修改 role 欄位（不是在權限頁做，是在員工資料頁）
3儲存後，該員工的「角色預設」整個換成新角色的預設
注意： 改角色後，之前的「個別調整」不會自動清除 — 仍會疊加在新角色上。如果想完全乾淨，記得搭配「情境 C」清零。

6. 注意事項與常見問題

Q1：為什麼有些員工看不到「修改」按鈕，只有「查詢」？

因為這個功能本身只設計成「唯讀」或「可寫」其中一種權限。例如「薪資查看」可能只有查詢權限，沒有對應的修改權限。

Q2：簽核（HR 表單核准）的權限要怎麼設？

簽核權限不在這頁設定。系統用 簽核鏈（chain） 決定誰能簽：

  • 是不是該關卡的指定簽核人
  • 是不是 HR 組織圖上的對應主管（store manager / supervisor）

要調整簽核流程，請到「專案流程 → 簽核設定」修改 chain 的簽核步驟。

Q3：改了權限對方要重新登入嗎？

建議請該員工重新整理瀏覽器（或重登）— 權限是登入時抓取的，現場改了當下不會立刻反映。

Q4：admin 自己的權限可以被改嗎？

技術上可以，但**強烈不建議** — admin 把自己權限改沒了會鎖住自己。如果要做這種改動，務必先找 super_admin 確認備援。

Q5：個別調整的紀錄會留嗎？

會。每筆個別調整都有日期 — 在「手動調整」標籤上會顯示 MM/DD。歷史誰改了哪個權限，目前透過資料庫的 employee_permissions 表的 override_at 欄位追蹤。

建議： 個別調整不要設太多 — 太多個別調整代表「角色設定可能有問題」。如果同樣的個別調整反覆出現在多位員工身上，考慮直接修改該角色的預設權限。
